# sql-sales-and-customer-analytics
Sales and customer analytics project using SQL


## Domain
Business Analytics / Sales Analytics

## Project Overview
This project demonstrates the use of SQL for analyzing sales and customer data in a business context.  
A relational database schema was designed and analytical queries were written to extract actionable insights.

> **Note:** The dataset used is synthetically generated to simulate real-world sales data; all data modeling and analysis were performed using actual SQL queries.

---

## Objectives
- Analyze customer purchasing behavior
- Calculate revenue metrics
- Identify top-performing products
- Perform regional and monthly sales analysis
- Demonstrate advanced SQL techniques

---

## Tools & Technologies
- SQL (PostgreSQL / MySQL compatible)
- Relational Database Design

---

## Key SQL Concepts Used
- JOINs (INNER JOIN)
- Aggregations (GROUP BY, HAVING)
- Subqueries
- Common Table Expressions (CTEs)
- Window Functions
- Date-based analysis

---

## Project Structure
- `schema/` – Database table definitions  
- `data/` – Synthetic data insertion scripts  
- `queries/` – Analytical SQL queries  
- `notes/` – Additional explanations and assumptions  

---

## How to Run
1. Create a database in PostgreSQL/MySQL
2. Run `schema/create_tables.sql`
3. Run `data/insert_data.sql`
4. Execute queries from the `queries/` folder

- Monthly revenue trends
